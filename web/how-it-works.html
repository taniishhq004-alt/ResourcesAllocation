How It Works

Priority Queue & Max-Heap Data Structure

Data Structure Explanation

📊 What is a Priority Queue?

A Priority Queue is an abstract data type similar to a regular queue, but each element has a priority associated with it. In a priority queue, an element with high priority is served before an element with low priority — regardless of the order in which they were inserted.

In our Emergency Resource Allocation System, each emergency request has an urgency level (1-10). The request with the highest urgency is always allocated resources first, making a Priority Queue the perfect data structure for this problem.

🏗️ Max-Heap Implementation

We implement the Priority Queue using a Max-Heap — a complete binary tree where the value of each node is greater than or equal to the values of its children. This ensures the root always contains the maximum (highest urgency) element.

Key Properties

  • Complete Binary Tree — All levels are fully filled except possibly the last
  • Heap Property — Parent node ≥ Children nodes (for Max-Heap)
  • Array-Based — Stored in a simple array, no pointers needed
  • For node at index i: Parent = (i-1)/2, Left child = 2i+1, Right child = 2i+2

🔧 Core Operations

Operation Description Time Complexity
insert() Add a new emergency request to the queue O(log n)
extractMax() Remove and return the highest-urgency request O(log n)
heapifyUp() Restore heap property upward after insertion O(log n)
heapifyDown() Restore heap property downward after extraction O(log n)
peek() View highest-priority request without removing O(1)

🚦 System Workflow

Here's how the Emergency Resource Allocation System processes requests step by step:

1

📥 Add Emergency Request

The user enters details: location name, disaster type (flood, earthquake, fire, etc.), number of people affected, urgency level (1-10), and resources needed (food, medical, rescue).

2

📊 Insert into Priority Queue

The request is added to the Max-Heap array. The heapifyUp() operation is called to bubble the new element up to its correct position, maintaining the heap property.

3

🔝 Highest Priority at Root

After insertion, the Max-Heap guarantees that the request with the highest urgency level is always at the root (index 0) of the array.

4

🚀 Allocate Resources

When "Allocate Resources" is triggered, extractMax() removes the root element (highest urgency). The last element is moved to the root, and heapifyDown() restores the heap.

5

📜 Record in History

The allocated request is recorded in the allocation history with its order number and timestamp, providing a complete audit trail of all dispatched resources.

6

🔄 Repeat

The process repeats — new requests can be added at any time, and resources are always dispatched to the most critical location first.

📝 Algorithm: Insert (HeapifyUp)

heapifyUp — Pseudocode Algorithm 
function heapifyUp(heap, index):
    while index > 0:
        parent = (index - 1) / 2
        if heap[index].urgency > heap[parent].urgency:
            swap(heap[index], heap[parent])
            index = parent
        else:
            break

📝 Algorithm: ExtractMax (HeapifyDown)

heapifyDown — Pseudocode Algorithm 
function heapifyDown(heap, index):
    largest = index
    left  = 2 * index + 1
    right = 2 * index + 2

    if left < size AND heap[left].urgency > heap[largest].urgency:
        largest = left
    if right < size AND heap[right].urgency > heap[largest].urgency:
        largest = right

    if largest ≠ index:
        swap(heap[index], heap[largest])
        heapifyDown(heap, largest)  // recursive call
