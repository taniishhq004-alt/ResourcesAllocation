About The Project

Emergency Resource Allocation System

BCA 2nd Semester · Minor Project

📌 Introduction

Natural disasters such as floods, earthquakes, fires, and accidents often create situations where multiple locations require emergency assistance at the same time. During such scenarios, resources like food, medical supplies, and rescue teams are limited. Therefore, it becomes essential to allocate these resources efficiently and fairly.

This project presents a simulation of an Emergency Resource Allocation System that demonstrates how data structures can be used to support decision-making during disaster situations. The system uses a Priority Queue to ensure that areas with higher urgency receive resources first. The project focuses on logical implementation and academic learning rather than real-time deployment.

⚠️ Problem Description

In disaster situations, emergency requests are received from different locations simultaneously. Each location may have a different level of urgency depending on factors such as the number of people affected and the severity of the situation. Without a proper prioritization mechanism, critical areas may experience delays in receiving help, leading to loss of life and resources.

The problem addressed by this project is to design a system that can allocate limited emergency resources in a structured and efficient manner. The system should be able to identify high-priority areas and ensure fair allocation using appropriate data structures. This project simulates such a system using priority-based logic.

✨ Key Features

📥
Accept Requests
Accept emergency requests from multiple disaster-affected locations with full details.
🔢
Urgency Levels
Assign urgency levels from 1 (low) to 10 (critical) for each emergency request.
📊
Priority Queue
Store all requests using a Max-Heap Priority Queue data structure for efficient ordering.
🚀
Smart Allocation
Automatically allocate resources to the highest-priority request first, every time.
📋
History Tracking
Track and display the complete history of all resource allocations in order.
🌐
Dual Interface
Available as both a C console program and a modern web-based simulation.

📋 Functional Requirements

The system shall be able to:

  1. Accept emergency requests from multiple locations
  2. Allow the user to specify urgency levels for each request
  3. Store emergency requests using a Priority Queue
  4. Allocate resources to the highest priority request first
  5. Handle multiple requests efficiently
  6. Display the order in which resources are allocated

Non-Functional Requirements

The system should:

  1. Be easy to understand and use
  2. Provide quick response for allocation decisions
  3. Be reliable and consistent in output
  4. Run on a standard C compiler or web browser
  5. Be scalable to handle additional requests
